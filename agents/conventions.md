# Conventions

## Source of truth
Markdown files in this repository are canonical unless explicitly marked as generated.

## Note types
- `journal`: daily entries and logs
- `evergreen`: durable reusable knowledge
- `skill`: reusable workflows/checklists/prompts
- `project-note`: temporary active-work notes
- `capture`: fast raw notes in inbox

## File placement
- raw or incomplete thoughts -> `inbox/`
- durable knowledge -> `notes/`
- active project material -> `projects/`
- templates/reference material -> `references/`
- agent instructions -> `agents/`

## Writing rules
- Prefer one concept per evergreen note
- Use literal titles
- Prefer markdown links over app-specific links
- Keep frontmatter minimal
- Do not invent taxonomy unless needed

## Promotion rule
Promote a capture into an evergreen note or skill when it has been useful at least twice or would be painful to rediscover.
